添加多语言支持

  因为在美帝学习，考虑到日后找工作的问题，所以想给博客添加一个英语版本的支持，这样以后找工作写简历的时候可以加分。此部分参考文章：文章1和文章2。
  文章1中分别创建两个hexo环境，拷贝public的方式是我最终选择的方案，但教程中存在一些没有提到的地方，再次补充。选择此方案的原因在于i18n虽好，但是没办法做到标签的区分，点击英文的标签会跳转到中文的网页，未来可能造成困扰；又因为是依托github.io的资源托管，所以使用nginx多域名代理的方式更加复杂且互相调用会有意想不到的bug。

基本的思想

  在Github里，用户名.github.io这个Repository会直接跳转到绑定的域名，但是对其他的Repository，Github会把其他Repository相应的主页作为主域名的子目录的方式解析。一个简单的例子就是：
  我的中文博客主页在Github中是上传到thebigoranger.github.io中的，通过在github设置个人域名解析为blog.theseele.site，这样直接在地址栏输入blog.theseele.site即可访问中文网站。再在Github中创建一个新的Repository，暂且命名为en好了，通过之前所说的，Github会把en相应的主页作为主域名的子目录的方式解析，
即thebigoranger.github.io/en/。由于主io中的域名跳转设置，所以目录的跳转也是绑定的，意味着在地址栏中输入blog.theseele.site/en/也能访问相同的英文网页。
  剩下所需要做的就是在服务器中设置合理的deploy方法，实现合理的资源跳转。
